The RigaCap brand is the **editorial paper theme** used by the live site: paper `#F5F1E8` (cards `#FAF7F0`), ink `#141210`, claret `#7A2430` (light `#8F2D3D` accents, positive `#2D5F3F`), hairline rules `#DDD5C7`, serif display (Fraunces on web, Georgia in email/canvas), mono for data. The **navy/gold** palette (`#172554` / `#f59e0b`) is the dead pre-rebrand design — it survives only in legacy assets.

**Why:** Jun 11 2026 — the launch-card rebuild brief said "preserve the visual system," so the agent faithfully rebuilt all five cards in the dead navy/gold brand and Erik had to catch it: "ALL of the launch cards used the old, non-paper design... mark that memory and never do it again."

**How to apply:** Any NEW or REGENERATED visual asset (social cards, OG images, PDFs, email templates, diagrams) gets the paper theme, full stop. When a brief says "preserve the visual system," first CHECK whether that system matches the current brand — if the source asset is navy/gold, the job includes redesigning to paper, not preserving. Legacy navy/gold files still in the repo (e.g. some design/documents covers, old launch cards) should be converted whenever touched.

**Social-card font legibility (Jun 13 2026):** 1080×1350 cards display ~400px wide in phone feeds (0.37× scale), so any canvas text under ~28px renders <11px on screen = unreadable. Floor: eyebrows/labels ≥26px, footer/disclaimer ≥26px, URL ≥28px, table headers ≥24px, body/values ≥30px. Feedback was "impressive but hard to read." All future cards must clear this floor.
